# 기획 전환: "퇴근하고 뭐하지?"

> 2026-07-02 결정. 일자리(워크넷) 개인 발급 불가로 확인 → 큐레이션 대상을 **동네 문화·여가·활동**으로 전환.
> 원래 job-to-be-done("내 동네에서 지금 할 수 있는 게 뭐가 있지?")은 유지. 지역은 **서울/수도권 먼저**.

## 왜 바꾸는가

- **일자리 데이터 못 씀**: 워크넷 채용 목록/상세는 개인회원 불가(민간 직업정보제공사업 신고 필요).
- **청년 지원금은 차별화 아님**: 이미 온통청년·복지로 등 정부 사이트가 존재. 모퉁이가 이길 축 아님.
- **모퉁이다움 = 동네**: "퇴근하고 뭐하지?"는 정부 포털이 답 못 주는 질문. 하이퍼로컬 큐레이션이 진짜 가치.
- **데이터가 실재**: 문화·공연·전시·체육시설 공공데이터는 개인 즉시 발급 + **위경도 제공** → 이미 만든 haversine 거리 스코어링이 실제로 작동.

## 콜드스타트 전략 (3단)

1. **공공데이터** — 첫날부터 원픽이 채워진다(사용자 0명이어도 동작). 콜드스타트 해결.
2. **사용자 데이터 축적** — 저장·재진단·"갔다왔어요" 피드백으로 스코어링 재보정 + 제보형 콘텐츠(나눔·떨이 등) 확장.
3. **타이밍 광고** — 원픽 옆에 "정보로서의 제휴"(유료 클래스·동네 상점). 어그로·배너 아님(DESIGN 안티레퍼런스 준수).

## 새 카테고리 (기존 OpportunityCategory 재정의)

| 신규 category | 뜻 | 뒷받침 데이터 | 진단 연결 |
|---|---|---|---|
| `culture` | 공연·전시·문화행사 | 한눈에보는 문화정보(좌표○) | timeSlot |
| `class` | 원데이 클래스·공방·강좌 | 문화 API 교육/체험, 지자체 평생학습 | timeSlot, energy |
| `active` | 운동·산책·등산·클라이밍 | 공공체육시설, 둘레길(좌표○) | energy(방전/활동형) |
| `food` | 늦게까지 하는 밥집·카페 | 소상공인 상권정보(좌표○) | timeSlot(평일저녁) |
| `market` | 장터·플리마켓·팝업 | 지자체 전통시장/장터 | timeSlot(주말) |
| `side_job` | **퇴근후 파트/단기 부업** | 서울 일자리플러스센터(15047488) | timeSlot, energy |

> 초기 주력 = `culture` + `active` + `side_job` (좌표 있어 거리 스코어링 즉시 작동). `class`/`food`/`market`은 보조축.
> **부업(`side_job`) 재포함** — 단, "퇴근후 또는 주말에 할 수 있는 **파트/단기만**" (정규직 제외, 고용형태코드 필터). 이전 seoul-jobs 어댑터의 파트/단기 분류 로직 재활용.
> 지원금(`subsidy`)은 보조 탭으로 유지 가능하나 원픽 대상에서 내림.

## ★ 큐레이션의 두 축: 시간 × 공간 ("퇴근하고")

컨셉의 핵심. 부가 기능이 아니라 스코어링의 뼈대.

### 공간축 — 위치 앵커 2개 (집 + 회사)
`Location`(단일) → **`UserAnchors { home?, work? }`** 로 확장.
- 회사근처(퇴근 시작점) / 집근처(퇴근 도착점) 각각 반경 내 기회에 거리 가점.
- `scoreOpportunity`의 haversine을 두 앵커 중 **가까운 쪽 거리**로 계산(min).
- home/work 중 하나만 있으면 그것만으로 계산(하위 호환).

### 시간축 — 퇴근후 참여 가능성 가점
- 문화/시설 데이터의 `행사시간`·운영시간이 **퇴근후(18~22시)**와 겹치면 가점.
- 진단 `timeSlot`(평일저녁/주말/유연)과 결합: 평일저녁 선택 → 저녁 시간대 기회 가점, 주말 → 주말 개최 가점.
- **하드 제외 아님(가점)** — 시간 정보 부정확한 데이터로 좋은 기회를 놓치지 않기 위해.

### 스코어링 가중치 개정 (DEFAULT_WEIGHTS 재보정 대상)
`fit`(진단↔카테고리) · `distance`(2앵커 min) · `time`(신규, 퇴근후 겹침) · `difficulty`(energy) · `cost`(예산, 무료 가점).

### 1순위 = 서울시 문화행사 (OA-15486)
응답 필드가 Opportunity와 거의 1:1: `분류→category` · `공연/행사명→title` · `장소·위/경도→location.point` · `자치구→dongName` · `시작/종료일→기간·deadline` · `행사시간→시간축 가점` · `유무료·이용요금→estimatedIncomeKrw(0=무료)` · `홈페이지→ctaUrl`. **매일 1회 갱신 · 공공누리 1유형(상업이용 가능)**.

## 진단(4문항) 재해석 — 스키마 변경 없이 의미만 이동

기존 `DiagnosisAnswers`(jobGroup/timeSlot/energy/incomeGoal)를 여가 맥락으로 재사용:
- `timeSlot`(평일저녁/주말/유연) → **언제** 할지 (그대로 유효)
- `energy`(방전/보통/활동형) → **활동 강도** (전시관람 vs 클라이밍) — 매우 잘 맞음
- `jobGroup` → 관심사 프록시로 약하게 사용하거나, 관심 카테고리 문항으로 교체 검토
- `incomeGoal` → **예산대(무료/저렴/상관없음)**로 의미 교체 검토

> core `diagnosis.ts`는 다음 단계에서 여가 맥락 문항으로 재설계(별도 작업).
