# 인체공학 및 접근성

레이아웃 가이드라인, 사이즈 사양, 플랫폼별 패턴, 그리고 접근성 요구사항. 모든 디자인의 협상 불가능한 표준입니다.

---

## 터치 & 클릭 타겟

| 요소 | 최소 | 권장 | Apple HIG | Material |
|------|------|------|-----------|----------|
| 터치 타겟 | 44×44pt | 48×48pt | 44×44pt | 48×48dp |
| 클릭 타겟 (웹) | 24×24px | 32×32px | — | 24×24px |
| 타겟 간격 | 8px | 12px | 8pt | 8dp |

### 작은 비주얼, 큰 타겟
```
탭 영역이 크면 시각 요소는 작아도 됩니다:

┌─────────────────┐
│                 │
│    [아이콘]     │  ← 비주얼: 24px
│                 │  ← 탭 영역: 48px
└─────────────────┘
```

규칙: 탭 영역 크기는 시각적 픽셀이 아닌 논리적 포인트로 측정됩니다. 48pt 탭 영역 안의 24px 아이콘은 정확합니다.

## 모바일 인체공학

### 엄지 영역 (한손 사용)

```
┌─────────────────────┐
│   닿기 어려움        │  ← 상태 정보, 덜 사용되는 액션
│                     │
├─────────────────────┤
│                     │
│   편안함            │  ← 콘텐츠, 보조 액션
│                     │
├─────────────────────┤
│   쉬움 / 주요        │  ← 주요 액션, 핵심 내비게이션
└─────────────────────┘
```

### 배치 규칙
- **주요 액션**: 화면 하단 1/3
- **내비게이션**: 하단 바 (iOS/Android 표준)
- **파괴적 액션**: 쉽게 닿는 영역에 두지 말 것 (실수 활성화 방지)
- **자주 토글되는 컨트롤**: 엄지 호 내 (오른손은 가운데-오른쪽)

### 한손 디자인 가정
- 사용자의 다른 손은 사용 중이다 (가방, 지하철 손잡이, 아이 들기)
- 모바일에서는 도달성이 시각적 위계보다 더 중요하다
- 액션 선택에 상단 모달보다 하단 시트
- 플로팅 액션 버튼 (FAB): 우하단 모서리, 56dp

## 플랫폼별 패턴

### 안전 영역

#### iOS
```
┌─────────────────────────┐
│░░░░ 상태 바 ░░░░░░░░░░░░│  ← 54pt (다이내믹 아일랜드) / 47pt (노치) / 20pt (레거시)
├─────────────────────────┤
│                         │
│      안전 콘텐츠         │
│        영역             │
│                         │
├─────────────────────────┤
│░░░░ 홈 인디케이터 ░░░░░░│  ← 34pt 하단 인셋
└─────────────────────────┘
```

- 항상 `safeAreaInsets` 준수 — 인터랙티브 요소를 시스템 UI 뒤에 절대 배치 금지
- 다이내믹 아일랜드: 54pt 상단 인셋. 콘텐츠가 겹치면 안 됨.
- 홈 인디케이터: 34pt 하단 인셋. 탭 바는 자동으로 이를 고려.
- 가로 모드: 노치/아일랜드를 위한 좌/우 추가 인셋

#### Android
```
┌─────────────────────────┐
│░░░░ 상태 바 ░░░░░░░░░░░░│  ← 24dp 기본, 디바이스에 따라 다름
├─────────────────────────┤
│                         │
│      안전 콘텐츠         │
│        영역             │
│                         │
├─────────────────────────┤
│░░░ 내비게이션 바 ░░░░░░░│  ← 48dp (3버튼) / 16dp (제스처)
└─────────────────────────┘
```

- 제스처 내비게이션: 16dp 하단 인셋 (스와이프 바)
- 3버튼 내비게이션: 48dp 하단 인셋
- Edge-to-edge: 모든 안전 영역 처리에 `WindowInsets` 사용

### 플랫폼 내비게이션 기대치

| 패턴 | iOS 컨벤션 | Android 컨벤션 |
|------|-----------|---------------|
| 뒤로 내비게이션 | 왼쪽 가장자리 스와이프, 좌상단 뒤로가기 셰브론 | 시스템 뒤로 버튼/제스처 |
| 탭 바 | 하단, 최대 5개, 아이콘+라벨 | 하단 내비게이션, 최대 5개 |
| 주요 액션 | 인라인 버튼 또는 우상단 내비 바 | FAB (플로팅 액션 버튼) |
| 컨텍스트 메뉴 | 롱 프레스 → 팝업 메뉴 | 롱 프레스 → 팝업, 점 3개 오버플로우 |
| 당겨서 새로고침 | 스피너와 함께 네이티브 풀다운 | SwipeRefreshLayout |
| 검색 | 큰 제목이 접히고, 검색 바가 나타남 | 검색 아이콘 있는 상단 앱 바 |
| 모달 | 하단에서 슬라이드 업, 드래그하여 닫기 | 풀스크린 또는 하단 시트 |
| 알림 | 중앙 다이얼로그, 2버튼 | Material AlertDialog |
| 스와이프 액션 | 리스트 항목에서 액션을 위한 왼쪽 스와이프 | 액션을 위한 좌/우 스와이프 |
| 선택 | 탭하여 선택, 파란색 체크마크 | 체크박스 또는 라디오 버튼 |

### 웹 특화 패턴

| 요소 | 데스크톱 | 태블릿 | 모바일 |
|------|---------|--------|--------|
| 내비게이션 | 상단 바 또는 사이드바 | 상단 바 (접을 수 있는 사이드바) | 하단 바 또는 햄버거 |
| 주요 액션 | 인라인 버튼 | 인라인 버튼 | 고정 하단 버튼 또는 FAB |
| 폼 | 다중 컬럼 가능 | 단일 컬럼 | 단일 컬럼 |
| 호버 상태 | 인터랙티브 큐로 필수 | 드물게 (스타일러스만) | 없음 — 호버에 의존 금지 |
| 우클릭 | 컨텍스트 메뉴 사용 가능 | 해당 없음 | 해당 없음 |

## 반응형 브레이크포인트

| 브레이크포인트 | 너비 | 디바이스 카테고리 | 레이아웃 |
|--------------|------|----------------|---------|
| Mobile S | 320px | 구형 iPhone, 작은 Android | 단일 컬럼, 스택 |
| Mobile M | 375px | iPhone 13/14/15, 표준 Android | 단일 컬럼 |
| Mobile L | 428px | iPhone Pro Max, 큰 Android | 단일 컬럼, 더 넓은 마진 |
| Tablet | 768px | iPad Mini, 작은 태블릿 | 2 컬럼 가능 |
| Tablet L | 1024px | iPad Air/Pro | 2 컬럼, 사이드바 |
| Desktop | 1280px | 노트북, 작은 모니터 | 다중 컬럼, 사이드바 |
| Desktop L | 1440px+ | 큰 모니터 | 최대 콘텐츠 너비, 중앙 정렬 |

### 반응형 디자인 규칙
- **모바일 우선**: 375px로 디자인하고, 그 다음 확장
- **콘텐츠 너비 제한**: 읽기 콘텐츠에 최대 680-720px (최적 라인 길이: 50-75자)
- **터치 타겟은 44pt+ 유지** — 화면이 더 커도 태블릿에서
- **모바일에서 콘텐츠를 숨기지 마라** — 재구조화하되, 제거하지 마라
- **320px에서 테스트** — 여기서 깨진다면, 실제 디바이스에서도 깨질 것

## 응답 시간

| 임계값 | 사용자 인지 | 디자인 요구사항 |
|--------|-----------|---------------|
| <100ms | 즉각 | 버튼 시각 피드백 (누름 상태) |
| <300ms | 빠름, 인디케이터 불필요 | 전환, 마이크로 인터랙션 |
| <1000ms | 흐름 유지 | 페이지 로드, 폼 제출. 스켈레톤 선택. |
| 1-3s | 알아챌 만한 지연 | 스켈레톤 화면 또는 시머 필요 |
| 3-10s | 상당한 대기 | 컨텍스트가 있는 진행 인디케이터 ("메시지 로딩 중...") |
| >10s | 포그라운드에서 허용 안 됨 | 백그라운드 프로세스 + 완료 시 알림 |

### 피드백 규칙
- **항상**: 누름/클릭 시 100ms 이내 시각 변화 (색상 시프트, 스케일, 리플)
- **300ms-1s**: 스켈레톤/시머 고려
- **>1s**: 로딩 인디케이터 필요
- **>3s**: 설명 텍스트가 있는 진행 인디케이터
- **>10s**: 백그라운드로 이동, 완료 시 알림

## 시각적 간격

### 컴포넌트 간격 (내부)
| 토큰 | 값 | 용도 |
|------|-----|------|
| 2xs | 2px | 헤어라인 보더, 미묘한 구분선 |
| xs | 4px | 타이트한 그룹화 (아이콘 + 라벨) |
| sm | 8px | 그룹 내 관련 항목 |
| md | 12px | 기본 컴포넌트 패딩 |
| lg | 16px | 편안한 컴포넌트 패딩 |
| xl | 24px | 넉넉한 컴포넌트 패딩 |

### 레이아웃 간격 (섹션 간)
| 토큰 | 값 | 용도 |
|------|-----|------|
| xs | 16px | 관련 섹션 |
| sm | 24px | 기본 섹션 간격 |
| md | 32px | 별개 섹션 |
| lg | 48px | 주요 섹션 분리 |
| xl | 64px | 페이지 레벨 분할 |
| 2xl | 96px | 히어로/랜딩 페이지 섹션 간격 (웹) |

### 간격 규칙
- 일관된 토큰 사용 — 절대 임의의 값 (13px, 17px 등) 금지
- 관련 항목: 더 좁은 간격. 무관한 항목: 더 넓은 간격.
- 더 많은 여백 = 더 많은 인지된 품질 (Apple은 넉넉한 간격 사용)
- 모바일: 데스크톱 대비 한 단계 줄임 (lg → md)

## 접근성 요구사항

### 협상 불가 (WCAG 2.1 AA 최소)

| 요구사항 | 명세 | 테스트 방법 |
|---------|------|------------|
| 텍스트 명도 | 최소 4.5:1 (≥18pt 큰 텍스트는 3:1) | 명도 체커 도구 |
| UI 컴포넌트 명도 | 인접 색상 대비 최소 3:1 | 시각 검사 |
| 포커스 인디케이터 | 가시 외곽선 2px+, 명도 3:1 | 키보드 내비게이션 테스트 |
| 터치 타겟 | 최소 44×44pt | 레이아웃 검사 |
| 텍스트 리사이징 | 200% 줌에서 콘텐츠 사용 가능 | 브라우저 줌 테스트 |
| 키보드 액세스 | 모든 기능을 키보드로 | 모든 것을 탭으로 통과 |

### 색상
- **절대** 색상을 상태의 유일한 지표로 사용 금지
- 항상 짝지을 것: 아이콘, 텍스트 라벨, 패턴, 또는 위치 변경
- 적록색맹, 청황색맹 시뮬레이터로 테스트

```
❌ 빨강 = 에러, 초록 = 성공 (색상만)
✅ ⚠️ 빨간 텍스트 + 아이콘 = 에러, ✓ 초록 텍스트 + 아이콘 = 성공

❌ "파란 버튼을 클릭하세요" (지시가 색상에 의존)
✅ "계속 클릭하세요" (지시가 라벨에 의존)
```

### 포커스 상태
```css
/* 키보드 사용자를 위한 가시적 포커스 */
:focus-visible {
  outline: 2px solid var(--color-focus);
  outline-offset: 2px;
}

/* 마우스/터치 사용자에게 외곽선 제거 */
:focus:not(:focus-visible) {
  outline: none;
}
```

규칙:
- 포커스 인디케이터는 모든 배경에 대해 가시적이어야 함
- 포커스 순서는 시각적 순서와 일치해야 함 (DOM 순서 = 시각적 순서)
- 포커스 트랩 없음 (사용자는 항상 Tab으로 빠져나갈 수 있어야 함)
- 모달 열림: 포커스가 모달로 이동. 모달 닫힘: 포커스가 트리거로 돌아옴.

### 스크린 리더

- 모든 인터랙티브 요소는 접근 가능한 이름이 필요 (`aria-label` 또는 가시 텍스트)
- 이미지: 정보 이미지에 `alt` 텍스트, 장식에 `alt=""`
- 폼 입력: 연관된 `<label>` 요소 (플레이스홀더만이 아닌)
- 동적 콘텐츠: 업데이트에 `aria-live="polite"`, 에러에 `aria-live="assertive"`
- 헤딩: 적절한 위계 (h1 → h2 → h3, 레벨 건너뜀 없음)
- 리스트: 시맨틱 `<ul>`, `<ol>`, `<li>` 요소 사용
- 랜드마크: 페이지 구조에 `<main>`, `<nav>`, `<header>`, `<footer>`

```tsx
// 아이콘만 있는 버튼 — aria-label 필요
<button aria-label="다이얼로그 닫기">
  <CloseIcon aria-hidden="true" />
</button>

// 폼 필드 — 라벨 + 헬퍼 텍스트
<label htmlFor="email">이메일 주소</label>
<input id="email" type="email" aria-describedby="email-hint" />
<span id="email-hint">인증 링크를 보내드립니다</span>

// 동적 알림
<div aria-live="polite" role="status">
  {saveStatus === 'saved' && '변경사항 저장됨'}
</div>
```

### 키보드 내비게이션
- 모든 인터랙티브 요소는 Tab으로 도달 가능
- 논리적 탭 순서 (시각적 순서 = DOM 순서)
- Escape는 모달/드롭다운/메뉴를 닫음
- Enter/Space는 버튼과 링크를 활성화
- 화살표 키는 메뉴, 탭, 라디오 그룹, 슬라이더용
- Home/End는 리스트의 시작/끝

### 인지 접근성
- 평이한 언어 사용 (8학년 읽기 수준 목표)
- 모든 페이지/화면에서 일관된 내비게이션
- 예측 가능한 동작 (같은 트리거 = 어디서나 같은 결과)
- 예상치 못한 컨텍스트 변경 없음 (자동 리다이렉트, 자동 제출)
- 연장 옵션이 있는 타임아웃 경고
- 에러 메시지가 무엇이 잘못됐고 어떻게 고칠지 명확히 설명
- 기억에 의존하지 마라: 정보를 보여주고, 사용자가 기억하길 기대하지 마라

## 모션 & 애니메이션

### 감소된 모션
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

### 추가 사용자 환경설정
```css
/* 고대비 모드 */
@media (prefers-contrast: more) {
  /* 보더 너비 증가, 단색 배경 사용, 투명도 제거 */
}

/* 투명도 감소 */
@media (prefers-reduced-transparency: reduce) {
  /* 반투명 배경을 단색 배경으로 대체 */
}

/* 다크 모드 */
@media (prefers-color-scheme: dark) {
  /* 적절한 명도 비율의 다크 팔레트 사용 */
}
```

### 애니메이션 지속시간 가이드
| 지속시간 | 용도 |
|---------|------|
| 100-150ms | 마이크로 인터랙션 (호버, 누름, 토글) |
| 200-300ms | 컴포넌트 전환 (펼침, 접힘, 페이드) |
| 300-500ms | 페이지/화면 전환, 모달 열기/닫기 |

### 이징
| 이징 | 용도 |
|------|------|
| ease-out | 뷰에 들어오는 요소 (빠른 시작, 부드러운 정지) |
| ease-in | 뷰에서 나가는 요소 (부드러운 시작, 빠른 출구) |
| ease-in-out | 상태가 변하는 요소 (위치, 크기) |
| linear | UI 전환에 절대 사용 금지 (기계적 느낌) |

### 피해야 할 것
- 깜박이는 콘텐츠 (초당 3회 이상) — 발작 위험
- 일시정지 컨트롤 없이 자동 재생 비디오/애니메이션
- 전정계 불편을 일으키는 패럴랙스 효과
- 콘텐츠에서 주의를 빼앗는 연속 루프 애니메이션
- 데이터 입력 중 애니메이션 (포커스 방해)

## 체크리스트

### 레이아웃
- [ ] 터치 타겟 ≥44×44pt (48×48pt 권장)
- [ ] 주요 액션이 엄지 영역에 (모바일)
- [ ] 안전 영역 준수 (노치, 다이내믹 아일랜드, 홈 인디케이터)
- [ ] 디자인 토큰을 사용한 일관된 간격
- [ ] 시각적 위계가 태스크 중요도와 일치
- [ ] 반응형: 320px 및 1440px+에서 테스트됨
- [ ] 읽기를 위해 콘텐츠 너비 720px로 제한

### 타이포그래피
- [ ] 본문 텍스트 ≥16px (웹) / 17pt (iOS) / 14sp (Android)
- [ ] 다이내믹 타입 / 확장 가능한 텍스트 지원
- [ ] 본문에 줄 높이 1.4-1.6배
- [ ] 최대 접근성 텍스트 크기로 테스트됨

### 접근성
- [ ] 명도 비율 통과 (텍스트 4.5:1, UI 3:1)
- [ ] 포커스 상태 가시 (외곽선 2px+, 명도 3:1)
- [ ] 색상이 상태의 유일한 지표가 아님
- [ ] 키보드 내비게이션 완료 (모든 기능 액세스 가능)
- [ ] 모든 인터랙티브 요소에 스크린 리더 라벨 존재
- [ ] 감소된 모션 준수 (`prefers-reduced-motion`)
- [ ] 폼 라벨 가시 (플레이스홀더만 아님)
- [ ] 에러 메시지 액세스 가능하고 설명적
- [ ] 헤딩 위계 정확 (레벨 건너뜀 없음)
- [ ] 시맨틱 HTML 랜드마크 사용
