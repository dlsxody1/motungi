# 3D 인터페이스 디자인

화면 매개 3D 경험을 위한 UX 원칙. 사용자는 2D 뷰포트 — 데스크톱 브라우저, 모바일 브라우저, 또는 임베디드 뷰어를 통해 3D 콘텐츠를 보고 조작합니다. 사용자는 항상 3D 씬 *바깥*에 있으며, 가상 카메라를 제어합니다.

기술적 구현 (렌더러, 프레임워크, API, 성능 최적화) 은 `web3d` 스킬을 참조하세요.

---

## 핵심 인터랙션 패러다임

> 화면은 3D 세계로의 창문이지, 사용자가 거주하는 공간이 아니다. 이는 사용자가 공간 *안에* 있는 XR과 근본적으로 다르다. 체화된 공간 디자인은 `xr-design.md` 참조.

### 2D에서 변하는 것

| 2D 화면 | 3D 뷰포트 |
|---------|-----------|
| 고정된 평면 레이아웃 | 깊이 축 — 객체가 다양한 거리에 존재 |
| 콘텐츠 노출을 위해 스크롤 | 탐험을 위해 궤도, 팬, 줌 |
| 클릭/탭으로 인터랙션 | 카메라를 통한 포인팅으로 3D 객체 선택 |
| 시각적 레이어링을 위한 z-index | 3D 공간의 실제 깊이 |
| 단일 시청 각도 | 사용자 제어 카메라 각도 |
| 호버 상태 (마우스) | 3D 표면의 교차 강조 |

### 그대로 유지되는 것

모든 2D UX 기본은 여전히 적용되며 — 깊이 축을 추가할 때 더욱 중요해집니다.
- 인지 부하 한계 (힉의 법칙, 밀러의 법칙)
- 컨텍스트당 하나의 주요 액션
- 모든 사용자 액션에 대한 피드백
- 플랫폼 컨벤션과의 일관성
- 에러 예방 및 복구
- 접근성은 선택이 아니다

## 카메라 인터랙션 디자인

카메라는 사용자가 3D 콘텐츠를 탐색하기 위한 주요 도구입니다. 빈약한 카메라 컨트롤은 다른 모든 것을 무관하게 만듭니다.

### 기대되는 컨트롤

**데스크톱 (마우스 + 키보드):**

| 액션 | 입력 | 동작 |
|------|------|------|
| 궤도 (회전) | 좌클릭 + 드래그 | 포커스 포인트 주위로 회전 |
| 줌 | 마우스 휠 / 트랙패드 핀치 | 포커스 포인트로 가까이/멀리 이동 |
| 팬 | 우클릭 + 드래그 / 중간 클릭 + 드래그 | 뷰 시프트 |
| 선택 | 객체에 좌클릭 | 3D 객체 선택 |
| 뷰 리셋 | 더블 클릭 / R 키 | 초기 카메라 위치로 돌아옴 |

**모바일 (터치):**

| 액션 | 제스처 | 동작 |
|------|------|------|
| 궤도 | 한 손가락 드래그 | 객체 주위로 회전 |
| 줌 | 두 손가락 핀치 | 줌 인/아웃 |
| 팬 | 두 손가락 드래그 | 뷰 시프트 |
| 선택 | 객체에 탭 | 3D 객체 선택 |
| 리셋 | 더블 탭 | 초기 뷰로 돌아옴 |

### 카메라 디자인 원칙

1. **사용자가 카메라를 제어한다.** 자동 카메라 움직임은 초기 진입 애니메이션으로만 허용됩니다. 그 후에는 100% 사용자 제어.
2. **움직임이 자연스럽게 느껴져야 한다.** 사용자가 놓을 때, 카메라는 부드럽게 감속해야 한다 (댐핑). 즉각 정지는 기계적이고 거슬리게 느껴진다.
3. **경계를 설정하라.** 카메라가 객체 안으로 들어가거나, 지면 아래로 내려가거나, 컨텍스트가 사라질 만큼 멀리 줌 아웃하는 것을 방지하라.
4. **리셋은 항상 사용 가능해야 한다.** 사용자는 언제든 "홈 뷰"로 돌아갈 방법이 필요하다 (더블 클릭/더블 탭).
5. **인터랙티브함을 신호하라.** 데스크톱에서는 3D 캔버스에서 커서를 grab/grabbing으로 변경. 모바일에서는 미묘한 어포던스 힌트 사용.

### 초기 카메라 위치

사용자가 콘텐츠를 가장 빨리 이해하도록 돕는 것에 따라 선택하세요.

| 콘텐츠 유형 | 카메라 위치 | 이유 |
|-----------|-----------|------|
| 제품 뷰어 | 3/4 뷰 (약간 위, 약간 각도) | 각도당 정보 최대화 |
| 건축 / 공간 | 시선 레벨, 입구를 바라봄 | 인간 관점과 일치 |
| 데이터 시각화 | 45° 위, 전체 데이터셋 표시 | 개요 우선, 필요 시 세부 |
| 캐릭터 / 아바타 | 상체 중앙, 약간 위 | 자연스러운 대화 각도 |
| 포토리얼리스틱 씬 (스플랫) | 가장 잘 캡처된 시점 | 아티팩트 최소화, 품질 최대화 |

### 카메라 전환

- 뷰 프리셋 간: 이징과 함께 0.5–1초
- 선택된 객체 포커스: 객체 중심을 향한 부드러운 줌
- **카메라를 즉시 텔레포트하지 마라** — 공간 인지를 파괴한다

## 3D 객체 인터랙션

### 인터랙션 패턴

| 인터랙션 | 데스크톱 | 모바일 | 피드백 |
|---------|---------|--------|--------|
| 선택 | 클릭 | 탭 | 외곽선 강조 + 미묘한 오디오 |
| 호버/포커스 | 마우스 호버 | 해당 없음 (탭 사용) | 외곽선 글로우 또는 머티리얼 시프트 |
| 잡기 | 클릭 + 드래그 | 롱 프레스 + 드래그 | 객체가 커서 따라옴 + 그림자 업데이트 |
| 회전 | 회전 핸들 드래그 | 두 손가락 트위스트 | 모멘텀과 함께 부드러운 회전 |
| 스케일 | 객체에 스크롤 또는 핸들 드래그 | 객체에 두 손가락 핀치 | 비례 리사이즈 + 스냅 가이드 |
| 배치 | 드래그 + 놓기 | 드래그 + 놓기 | 표면/그리드에 스냅 + 안착 애니메이션 |

### 피드백 규칙

- 모든 인터랙션은 즉각적인 시각 응답이 필요 (<100ms)
- 선택 상태는 명백해야 한다: 외곽선, 글로우, 또는 머티리얼 변경
- 오디오 피드백은 공간 인지를 강화한다 — 객체 위치에 국지화된 사운드
- 데스크톱: 모든 인터랙티브 객체에 호버 상태
- 모바일: 호버가 존재하지 않으므로 탭 어포던스 (미묘한 애니메이션, 시각적 힌트)

### 히트 영역 사이징

- 3D 객체 히트 영역은 시각적 표현보다 **더 커야** 함
- 모바일: 히트 볼륨을 적어도 44×44px 화면 투영을 커버하도록 확장
- 작은 객체는 더 쉬운 타겟팅을 위해 부풀린 보이지 않는 히트 볼륨 필요

## 시각적 위계로서의 깊이

z-index가 추상적인 2D와 달리, 3D 깊이는 문자 그대로이며 인지에 영향을 미칩니다.

```
카메라에 가장 가까움 (전경):
  → 활성 컨트롤, 선택된 객체, 모달 오버레이
  → 절제하여 사용 — 전경 혼잡은 압도적임

중간 거리 (주요):
  → 메인 콘텐츠, 주요 객체, 작업 공간
  → 예측 가능성을 위해 인터랙티브 콘텐츠를 일관된 깊이로 유지

먼 곳 (배경):
  → 환경적 컨텍스트, 비인터랙티브 참조 객체
  → 공간적 그라운딩과 분위기를 확립
```

**규칙**: 인터랙티브 요소 간 큰 깊이 점프를 피하라. 일관된 깊이는 눈 재포커싱 부담과 인지 부하를 줄인다.

### 그림자와 그라운딩

- **그림자는 필수다.** 그림자 없는 객체는 분리되어 떠 있는 것처럼 보인다.
- 지면 평면, 접촉 그림자, 또는 앰비언트 오클루전이 공간적 그라운딩을 제공한다.
- 가상 조명을 예상되는 컨텍스트와 일치시켜라 (제품에는 중립 스튜디오, 건축에는 환경적).

## 발견 가능성 및 첫 사용 경험

3D 뷰어는 발견 가능성 도전을 제시합니다 — 사용자는 인터랙션할 수 있다는 것을 깨닫지 못하거나, 어떤 인터랙션이 가능한지 모를 수 있습니다.

### 온보딩 패턴

| 패턴 | 사용 시점 |
|------|---------|
| **"드래그하여 회전" 힌트** | 어떤 3D 뷰어든 첫 방문. 3-5초 동안 표시, 첫 인터랙션 시 닫기. |
| **미묘한 자동 회전** | 5초 이상 유휴 시. 느린 속도. 사용자 인터랙션 시 즉시 정지. |
| **펄싱 핫스팟** | 클릭 가능한 부분이 있는 컨피규레이터. 인터랙션이 가능한 곳을 표시. |
| **가이드 투어** | 복잡한 3D 씬 (건축 워크스루). 주석과 함께 단계별 카메라 움직임. |

### 디자인 규칙

- 사용자가 이전에 3D 뷰어를 사용한 적이 없다고 가정하라
- 2D 컨트롤 (버튼, 메뉴) 을 3D 뷰포트와 함께 배치하라 — 컨트롤을 3D 씬 안에 숨기지 마라
- 뷰어가 무엇을 보여주는지 라벨링하라: "360° 제품 뷰" 또는 "인터랙티브 평면도"가 기대치를 설정한다
- 같은 정보에 대한 3D가 아닌 대안으로 텍스트 설명이나 사양 테이블을 제공하라

## 제품 컨피규레이터 UX

### 핵심 원칙

1. **즉각적인 피드백**: 옵션 변경은 3D 모델에 즉시 반영되어야 한다. >300ms 지연은 느리게 느껴진다.
2. **선택지 제한**: 카테고리당 4–8개 옵션. 과도한 선택은 의사결정 마비를 일으킨다 (힉의 법칙).
3. **가이드 플로우**: 단계별 진행 (모양 → 색상 → 머티리얼 → 액세서리). 자유 탐색 허용하되 권장 순서 제공.
4. **실시간 가격 업데이트**: 옵션 변경 시 가격이 즉시 변경. 체크아웃에서 사용자를 놀라게 하지 마라.
5. **모바일 우선**: 컨피규레이터 사용자의 55–80%가 모바일에서.

### 레이아웃 패턴

**데스크톱:**
```
┌───────────────────────────────────────────────┐
│  [ 3D 뷰어 (60–70% 너비) ] │ [ 옵션 패널 ]    │
│                            │ - 카테고리        │
│    궤도 / 줌 / 팬          │ - 색상 스와치    │
│                            │ - 머티리얼       │
│                            │ - 가격           │
└───────────────────────────────────────────────┘
```

**모바일:**
```
┌───────────────────────┐
│   [ 3D 뷰어 50–60% ] │
│   궤도 / 줌 / 핀치     │
├───────────────────────┤
│  [ 탭: 카테고리 ]      │
│  [ 옵션 그리드/스와치 ] │
│  [ 고정 하단: 가격 ]   │
└───────────────────────┘
```

### 핵심 인터랙션

- **색상 스와치**: 탭하면 3D 모델의 머티리얼이 즉시 교체
- **핫스팟**: 모델 표면의 클릭 가능한 포인트 — "이 부분 커스터마이즈"
- **비교 모드**: "현재 vs 수정" 분할 뷰 또는 토글
- **공유 가능한 상태**: URL에 인코딩된 컨피규레이션 — 링크 공유로 정확한 상태 재현
- **스냅샷**: 현재 컨피규레이션의 스크린샷 저장

## 포토리얼리스틱 씬 콘텐츠 (가우시안 스플래팅)

가우시안 스플래팅은 실세계 사진에서 캡처된 포토리얼리스틱 씬을 렌더링합니다. 전통적인 3D 모델 (폴리곤 + 텍스처) 과 달리, 스플랫 씬은 거의 사진 같은 느낌을 줍니다 — 이는 사용자 기대치를 변화시킵니다.

### 사용 시점

| 사용 사례 | 스플랫이 작동하는 이유 |
|---------|---------------------|
| 부동산 워크스루 | 휴대폰 사진에서 포토리얼리스틱 방 캡처 |
| 이커머스 제품 스캔 | 수동 3D 모델링 없이 실물 그대로의 외관 |
| 문화 유산 / 박물관 | 높은 충실도로 실세계 씬 보존 |
| 관광 / 위치 미리보기 | 사진 캡처에서 내비게이션 가능한 씬 |

### 전통적 3D와의 UX 차이

- **사용자가 사진 품질을 기대한다.** 씬 경계나 언더샘플된 영역의 아티팩트는 "깨진" 콘텐츠처럼 느껴진다 — 가장자리 영역을 자르거나 마스크하라.
- **내비게이션이 제한되어야 한다.** 스플랫 씬은 정의된 카메라 경로에서 가장 잘 작동한다. 자유 로밍은 관찰되지 않은 각도에서 렌더링 아티팩트를 노출한다.
- **로딩이 더 무겁다.** 스플랫 파일은 크다 (50–200MB). 점진적 로딩 (먼저 저밀도 미리보기, 그 다음 디테일 스트리밍) 이 필수.
- **폴백이 중요하다.** 스플랫을 렌더링할 수 없는 디바이스를 위해 사전 렌더된 이미지 갤러리나 360° 사진 뷰어를 제공하라.

## 스크롤 기반 3D 스토리텔링

3D 애니메이션을 제어하는 스크롤 입력은 매력적인 내러티브를 만든다 — 제품 공개, 데이터 스토리, 공간적 설명. 이는 자유 궤도 3D 뷰어와 별개의 패러다임이다.

### 디자인 규칙

1. **진행 인디케이터는 필수다.** 사용자는 시퀀스에서 어디에 있는지 알아야 한다 (점, 바, 또는 퍼센트).
2. **건너뛰기 컨트롤은 필수다.** 사용자는 언제든 앞으로 점프하거나 시퀀스를 탈출할 수 있어야 한다.
3. **감소된 모션 환경설정 준수.** 애니메이션 없이 키 프레임을 보여주는 정적 폴백 제공.
4. **3D 없이도 콘텐츠가 작동해야 한다.** 텍스트, 핵심 정보, CTA는 애니메이션과 독립적으로 액세스 가능해야 한다.
5. **짧게 유지하라.** 3–5개의 "씬" 또는 단계. 긴 시퀀스 (>10) 는 피로와 이탈을 일으킨다.
6. **스크롤을 가로채지 마라.** 3D 경험은 사용자가 다른 콘텐츠에 도달하기 위해 그것을 지나 스크롤하는 것을 막아서는 안 된다.

## 로딩 경험

3D 자산은 무겁습니다. 사용자는 빈 화면을 응시하며 3초 이상 기다리지 않습니다.

### 로딩 위계

```
1. 즉각 (0ms)
   └── 컨테이너 + 스켈레톤/플레이스홀더 즉시 가시

2. 빠른 미리보기 (0–2s)
   └── 포스터 이미지 또는 초저폴리 실루엣

3. 점진적 (2–10s)
   └── 기본 지오메트리 먼저 → 그 다음 텍스처 로드
   └── 사용자가 로딩 중에 이미 궤도/줌 가능

4. 풀 퀄리티 (10s+)
   └── 고해상도 텍스처가 백그라운드에서 스왑
   └── 가시 팝인 최소화
```

### 로딩 UI 패턴

| 패턴 | 최적 용도 |
|------|---------|
| 포스터 이미지 + 진행 바 | 단일 제품 뷰어 |
| 스켈레톤 플레이스홀더 | 더 큰 페이지 내 3D 영역 |
| 저해상도 → 고해상도 스왑 | 큰 씬 (가상 투어, 스플랫) |
| 스크롤 시 레이지 로드 | 폴드 아래의 3D 콘텐츠 |

### 핵심 규칙

> 로딩 중에 인터랙션을 허용하라. 저폴리 모델이 준비되면, 즉시 궤도/줌을 활성화하라. 절대 로딩 오버레이로 인터랙션을 차단하지 마라.

---

## 모바일 UX

- **스크롤 충돌**: 3D 캔버스의 터치는 페이지 스크롤과 충돌한다. 3D 인터랙션 영역을 스크롤 가능 영역에서 명확히 분리하라.
- **작은 터치 타겟**: 3D 객체 탭 히트 영역을 적어도 44×44px 화면 투영으로 부풀려라.
- **성능**: 고폴리 모델은 모바일에서 프레임을 떨어뜨린다. 매끄러운 인터랙션을 유지하기 위해 경험은 자동으로 품질을 조정해야 한다.
- **배터리**: 사용자가 인터랙션하지 않을 때 렌더 루프를 일시 정지하라.
- **화면 비율**: 세로 모드에서, 3D 뷰어는 화면 높이의 ≤60%를 차지해야 한다. 남은 공간에 2D 컨트롤을 배치하라.

---

## 점진적 경험 레벨

3D 경험을 단일 플랫폼 베팅이 아닌 계층화된 능력으로 디자인하세요.

```
레벨 0: 정적 폴백
├── 고해상도 렌더 이미지 또는 360° 이미지 뷰어
└── 대상: 미지원 브라우저, 접근성, 초저사양 디바이스

레벨 1: 인라인 3D (핵심 경험)
├── 궤도, 줌, 팬이 있는 브라우저 3D 뷰어
└── 대상: 대부분의 현대 브라우저 (기본 경험)

레벨 2: 몰입 (선택) — xr-design.md 참조
├── 모바일 AR 배치 또는 VR 헤드셋 세션
└── 대상: 호환 디바이스 + 명시적 사용자 옵트인
```

**규칙:**
- 레벨 1은 완전한 경험이어야 한다. AR/VR은 보너스이며, 결코 요구사항이 아니다.
- 미지원 기능에 대해 절대 에러 메시지를 표시하지 마라. 디바이스가 AR을 할 수 없다면, AR 버튼을 표시하지 마라. 사용자의 디바이스를 탓하는 것은 빈약한 UX다.
- 작동하는 것만 표시하라.

## 3D 접근성

### 키보드 내비게이션

3D 뷰어는 키보드 액세스 가능해야 함:
- Tab으로 뷰어에 포커스
- 화살표 키로 궤도
- +/- 또는 PgUp/PgDn으로 줌
- Enter로 객체 선택
- Escape로 포커스 해제

### 대안 표현

- 3D 뷰어와 함께 텍스트 설명이나 사양 테이블 제공
- 주요 상태 변경을 스크린 리더에 알림 ("색상이 빨강으로 변경됨", "정면에서 보는 중")
- 3D 뷰어는 향상이다 — 그것이 전달하는 정보도 텍스트로 사용 가능해야 한다

### 모션 민감성

- 감소된 모션 환경설정 준수: 자동 회전 비활성화, 카메라 애니메이션 최소화
- 파티클 효과와 무한 루프 애니메이션 비활성화
- 사용자 시작 모션은 허용; 시스템 시작 모션은 안 됨

### 색상과 시각

- 색상 스와치는 텍스트 라벨을 포함해야 함 (색상만이 아닌)
- 3D 모델의 시각적 구분이 색상에만 의존해서는 안 됨
- 저시력 사용자를 위한 충분한 줌 범위 제공
- UI 오버레이에 고대비 모드 지원

## 3D를 사용하지 말아야 할 때

다음의 경우 2D에 머무르세요:

- 태스크가 주로 텍스트 중심 (읽기, 쓰기, 데이터 입력)
- 공간적 이해보다 실행 속도가 더 중요
- 데이터가 본질적으로 표 형식, 순차적, 또는 평면적
- 공간 차원이 기능적 가치를 추가하지 않음

**규칙**: 3D는 2D가 할 수 없는 문제를 해결해야 한다. 주요 가치가 "멋져 보임"이라면, 2D에 머물러라.

## 안티 패턴

| 안티 패턴 | 실패 이유 | 대안 |
|----------|---------|------|
| 3D가 완전히 로드될 때까지 빈 화면 | 3+초 빈 화면 → 50%+ 이탈률 | 포스터 이미지 + 진행 인디케이터 |
| 모바일과 데스크톱에 동일한 품질 모델 | 프레임 드롭, 배터리 소모, 과열 | 적응형 품질, 모바일용 별도 LOD |
| 3D 캔버스가 전체 뷰포트 채움 (모바일) | 스크롤 불가, 다른 콘텐츠 액세스 불가 | 캔버스를 화면 높이의 50–60%로 제한 |
| AR 미지원 시 에러 메시지 | 사용자의 디바이스를 탓함 | AR 버튼을 완전히 숨김 |
| 자동 재생 3D 애니메이션 + 사운드 | 예상치 못한 모션/사운드가 컨텍스트 방해 | 사용자 인터랙션 시 시작, 사운드 기본 끔 |
| 3D 씬 안에만 있는 컨트롤 | 발견 어려움, 액세스 불가 | 2D 컨트롤을 3D 뷰어와 함께 배치 |
| 모바일에서 터치 핸들링 없음 | 페이지 스크롤이 3D 인터랙션에 가로채임 | 인터랙션 영역을 명확히 분리 |
| 3D 콘텐츠에 텍스트 대안 없음 | 접근성 위반, 정보가 시각만의 형식에 잠김 | 뷰어와 함께 텍스트 설명/사양 |
| 즉각적인 카메라 텔레포트 | 공간적 컨텍스트와 방향 파괴 | 이징과 함께 부드러운 전환 |
| 2D로 충분할 때 3D 사용 | 더 느림, 더 복잡, 추가 가치 없음 | 3D가 2D가 할 수 없는 문제를 해결할 때만 |
| 포토리얼리스틱 씬에서 무제한 내비게이션 | 가장자리 아티팩트가 품질 환상 파괴 | 카메라 경로 제한, 가장자리 마스크 |
| 건너뛰기/탈출 없는 스크롤 기반 3D | 사용자가 건너뛸 수 없는 애니메이션에 갇힌 느낌 | 건너뛰기 컨트롤 + 섹션으로 점프 |
| 감소된 모션 폴백 없는 스크롤 기반 3D | 접근성 위반 | 정적 키 프레임 폴백 |

## 검증 체크리스트

### 로딩
- [ ] 3D 로드 전 포스터/스켈레톤 표시됨
- [ ] 3초 이내 첫 의미 있는 렌더
- [ ] 로딩 중 인터랙션 가능 (저폴리에서 궤도/줌)
- [ ] 매끄러운 품질 향상 (거슬리는 팝인 없음)

### 카메라 & 인터랙션
- [ ] 데스크톱 (궤도/줌/팬) 과 모바일 (드래그/핀치) 모두 작동
- [ ] 카메라 움직임에 부드러운 댐핑 — 즉각 정지 없음
- [ ] 뷰 리셋 사용 가능 (더블 클릭/더블 탭)
- [ ] 카메라가 객체 안이나 지면 아래로 갈 수 없음
- [ ] 모바일에서 스크롤과 3D 인터랙션 충돌 없음

### 발견 가능성
- [ ] 사용자가 인터랙션 가능함을 이해 (힌트, 라벨, 또는 어포던스)
- [ ] 2D 컨트롤이 3D 뷰어와 함께 배치 (안에 숨겨지지 않음)
- [ ] 콘텐츠 유형 라벨링됨 ("360° 뷰", "인터랙티브 평면도")

### 점진적 경험
- [ ] 미지원 브라우저용 정적 이미지 폴백
- [ ] 미지원 시 AR/VR 버튼 숨김 (에러 메시지 아님)
- [ ] AR/VR 없이도 핵심 경험 작동

### 접근성
- [ ] 키보드 내비게이션 작동
- [ ] 3D 콘텐츠에 텍스트 대안 제공
- [ ] 감소된 모션 환경설정 준수 (자동 회전 없음)
- [ ] 색상 선택에 텍스트 라벨 포함
- [ ] 상태 변경에 스크린 리더 알림

### 컨피규레이터 (해당 시)
- [ ] 옵션 변경이 모델에 <300ms 내 반영
- [ ] 카테고리당 4–8개 옵션 (선택 과부하 없음)
- [ ] 실시간 가격 업데이트
- [ ] URL을 통한 컨피규레이션 공유 가능
- [ ] 모바일 레이아웃 테스트됨 (뷰어 + 컨트롤, 스크롤 충돌 없이)

### 포토리얼리스틱 씬 (해당 시)
- [ ] 미리보기와 함께 점진적 로딩
- [ ] 카메라 경로가 캡처된 시점으로 제한
- [ ] 가장자리 아티팩트 마스크 또는 자름
- [ ] 미지원 디바이스 폴백

### 스크롤 기반 3D (해당 시)
- [ ] 진행 인디케이터 가시
- [ ] 건너뛰기 / 점프 컨트롤 사용 가능
- [ ] 감소된 모션 폴백 작동
- [ ] 3D 애니메이션 없이도 콘텐츠 기능적
