# 정보 구조 (Information Architecture)

정보 공간의 구조적 디자인. IA는 사용자가 콘텐츠를 어떻게 찾고, 이해하고, 이동할지를 결정합니다.

---

## 핵심 원칙

> IA는 잘 됐을 때 보이지 않습니다. 사용자는 구조에 대해 생각하지 않고도 필요한 것을 찾습니다.

목표는 콘텐츠를 논리적으로 정리하는 것이 아니라 — **사용자가 정리될 거라고 기대하는 방식으로** 콘텐츠를 정리하는 것입니다. 논리와 사용자 기대는 종종 갈라집니다.

---

## 댄 브라운의 IA 8가지 원칙

### 1. 객체 원칙 (Principle of Objects)
콘텐츠를 생애주기, 행동, 속성을 가진 살아있는 것으로 다루세요.
- "제품"은 단순한 페이지가 아니다 — 이름, 가격, 상태, 관계, 그리고 이력을 가진다
- 내비게이션을 디자인하기 전에 객체 모델을 디자인하라

### 2. 선택 원칙 (Principle of Choices)
의미 있는 선택지를 제시하세요. 더 많은 게 더 좋은 게 아니라 — 관련된 게 더 좋다.
- 각 내비게이션 레벨: 최대 5-9개 옵션 (밀러의 법칙)
- 모든 옵션은 형제 옵션과 명확히 구별되어야 한다
- 사용자가 라벨 뒤에 무엇이 있는지 예측할 수 없다면, 그 라벨은 실패한 것이다

### 3. 공개 원칙 (Principle of Disclosure)
사용자가 더 깊이 파고들 때 무엇을 찾을지 이해하도록 충분한 정보만 보여주세요.
- 점진적 공개: 요약 → 세부 → 전체 콘텐츠
- 각 레벨은 다음 질문에 답한다: "더 깊이 가고 싶은가?"
- 복잡성을 절대 처음부터 보여주지 마라

### 4. 예시 원칙 (Principle of Exemplars)
카테고리를 설명할 때 콘텐츠의 예시를 보여주세요.
- 노트북 썸네일이 있는 "전자제품"이 "전자제품"만 있는 것보다 명확하다
- 미리보기, 썸네일, 발췌는 내비게이션 불확실성을 줄인다

### 5. 정문 원칙 (Principle of Front Doors)
홈페이지가 아니라 어떤 페이지에든 사용자가 도착한다고 가정하세요.
- 모든 화면은 다음을 필요로 한다: 정체성(나는 어디에 있는가?), 방향(여기에 무엇이 있는가?), 내비게이션(어디로 갈 수 있는가?)
- 딥링크, 검색 결과, 그리고 공유된 URL은 모두 홈페이지를 우회한다

### 6. 다중 분류 원칙 (Principle of Multiple Classification)
같은 콘텐츠를 찾는 여러 방법을 제공하세요.
- 카테고리로 탐색 + 키워드로 검색 + 속성으로 필터
- 사용자는 다른 멘탈 모델을 가진다 — 모두 지원하라
- 예: 아티스트별, 장르별, 분위기별, 또는 활동별 음악

### 7. 집중 내비게이션 원칙 (Principle of Focused Navigation)
같은 시스템에서 다른 유형의 내비게이션을 섞지 마세요.
- 주 내비게이션: 최상위 섹션
- 로컬 내비게이션: 섹션 내 이동
- 유틸리티 내비게이션: 계정, 설정, 도움말
- 이것들을 단일 메뉴로 절대 섞지 마라

### 8. 성장 원칙 (Principle of Growth)
규모를 위해 디자인하세요. IA는 재구조화 없이 10배의 콘텐츠를 수용해야 합니다.
- 카테고리 이름은 콘텐츠가 성장해도 정확해야 한다
- 라벨에 구체적인 카운트를 피하라 ("3 카테고리"는 12가 되면 깨진다)
- 아직 존재하지 않는 콘텐츠 유형을 위해 계획하라

## 내비게이션 패턴

### 패턴 선택 가이드

| 사용자 태스크 | 최적 패턴 | 이유 |
|--------------|----------|------|
| 3-5개 최상위 섹션 사이 전환 | 탭 바 (모바일) / 상단 내비 (웹) | 지속적, 가시적, 한 탭 액세스 |
| 위계적 콘텐츠로 드릴다운 | 스택 내비게이션 | 깊이의 멘탈 모델, 쉬운 뒤로가기 |
| 컨텍스트를 떠나지 않고 유틸리티/설정 액세스 | 모달 / 하단 시트 | 오버레이가 컨텍스트를 보존 |
| 큰 화면에서 많은 카테고리 탐색 | 사이드바 / 드로어 | 확장 가능, 접을 수 있음 |
| 다단계 선형 태스크 완료 | 위저드 / 스테퍼 | 진행 가시적, 단계 강제 |

### 탭 바 (모바일)

```
┌─────────────────────────────────┐
│                                 │
│         콘텐츠 영역               │
│                                 │
├────┬────┬────┬────┬────────────┤
│ 🏠 │ 🔍 │ ➕ │ 💬 │ 👤        │
│ 홈 │ 검색│ 새로│ 채팅│ 프로필    │
└────┴────┴────┴────┴────────────┘
```

규칙:
- 최대 3-5개 항목 (절대 5개 초과 안 됨)
- 아이콘 + 라벨 항상 (아이콘만은 접근성 실패)
- 순서: 가장 많이 사용 → 가장 적게 사용, 왼쪽 → 오른쪽
- 현재 섹션을 명확하게 강조
- 미확인/대기 항목에 대한 배지 인디케이터

### 스택 내비게이션 (드릴다운)

```
[리스트] → [상세] → [하위 상세]
   ←  뒤로    ←  뒤로
```

규칙:
- 항상 뒤로가기 어포던스 표시 (셰브론 + 이전 제목)
- 제목은 섹션 이름이 아닌 현재 콘텐츠를 반영
- iOS의 스와이프 백 제스처 (왼쪽 가장자리에서 스와이프)
- 깊은 스택 (>3 레벨)은 IA를 평탄화해야 한다는 신호

### 하단 시트

```
┌─────────────────────────────────┐
│      부모 화면 (어두워짐)         │
├─────────────────────────────────┤
│  ━━━ (드래그 핸들)              │
│                                 │
│  시트 콘텐츠                     │
│  - 옵션 A                       │
│  - 옵션 B                       │
│  - 옵션 C                       │
│                                 │
└─────────────────────────────────┘
```

규칙:
- 컨텍스트 액션, 필터, 선택에 사용
- 드래그 핸들 항상 가시
- 외부 탭 또는 아래로 스와이프하여 닫기
- 세 개의 스냅 포인트: 살짝(25%), 절반(50%), 전체(90%)
- 절대 하단 시트를 중첩하지 마라

### 사이드바 (웹 / 태블릿)

```
┌──────────┬──────────────────────┐
│ 섹션 A   │                      │
│ 섹션 B   │    콘텐츠 영역        │
│ 섹션 C   │                      │
│ ──────── │                      │
│ 설정     │                      │
│ 도움말   │                      │
└──────────┴──────────────────────┘
```

규칙:
- 작은 뷰포트에서 접을 수 있음 (햄버거 트리거)
- 구분선이나 헤딩으로 관련 항목 그룹화
- 현재 섹션 강조
- 유틸리티 항목 (설정, 도움말)은 하단에, 분리됨
- 최대 깊이: 2 레벨 (섹션 → 하위 섹션)

## IA 디자인 프로세스

### Step 1: 콘텐츠 인벤토리
제품이 지원해야 할 모든 콘텐츠와 기능을 나열하세요.
- 페이지/화면, 기능, 설정, 도움말 콘텐츠
- 콘텐츠 유형: 정적, 동적, 사용자 생성, 시스템 생성
- 관계: 부모-자식, 형제, 교차 참조

### Step 2: 카드 정렬 (사용자 멘탈 모델 발견)
```
오픈 정렬:  사용자가 스스로 그룹을 만들고 이름을 지음
클로즈드 정렬: 사용자가 미리 정의된 카테고리로 카드를 정렬
하이브리드 정렬: 미리 정의된 카테고리 + 새로 만들 옵션
```
- 신뢰할 만한 패턴을 위해 최소 15명 참가자
- 살펴볼 것: 합의 (>60% 같은 그룹화)와 불일치 (모호함의 신호)
- 불일치를 사용해 다중 액세스 경로가 필요한 콘텐츠 식별

### Step 3: 위계 정의

```
레벨 0: 앱/사이트 (암묵적 — 사용자는 이미 "앱 안에" 있음을 안다)
레벨 1: 주 섹션 (탭 바 / 상단 내비 — 3-5개 항목)
레벨 2: 하위 섹션 또는 콘텐츠 리스트
레벨 3: 상세 보기
레벨 4+: 가능하면 피하라 — 평탄화하거나 검색 사용
```

규칙: 사용자가 핵심 콘텐츠에 도달하기 위해 >3 탭/클릭이 필요하다면, IA가 너무 깊다.

### Step 4: 트리 테스트 (검증)
- 사용자에게 태스크를 줘라: "알림 설정을 변경할 곳을 찾으세요"
- 측정: 성공률, 직접성 (첫 클릭이 정답?), 시간
- 통과 기준: >80% 성공률, >60% 직접성
- 실패: 재구조화하고 재테스트

### Step 5: 사이트맵 생성

```
[앱 루트]
├── 홈
│   ├── 피드
│   └── 추천
├── 검색
│   ├── 결과
│   └── 필터
├── 만들기
│   ├── 새 게시물
│   └── 임시 저장
├── 메시지
│   ├── 받은편지함
│   └── 스레드
└── 프로필
    ├── 설정
    ├── 계정
    └── 도움말
```

산출물 형식: 문서에 머메이드 다이어그램 또는 ASCII 트리.

## 검색 vs 탐색

| 신호 | 검색 선호 | 탐색 선호 |
|------|----------|----------|
| 콘텐츠 양 | >100개 항목 | <100개 항목 |
| 사용자가 원하는 것을 안다 | 예 — 구체적인 쿼리 | 아니오 — 탐험 중 |
| 콘텐츠가 동질적 | 예 (모두 제품, 모두 글) | 아니오 (혼합 유형) |
| 사용자 어휘가 콘텐츠와 일치 | 예 | 아니오 — 먼저 노출 필요 |

### 검색 베스트 프랙티스
- 검색 박스: 모든 화면에서 가시적 (또는 한 탭 거리)
- 2자 이상 입력 후 자동 제안
- 최근 검색 표시
- 오타 처리 (퍼지 매칭)
- 빈 결과: 대안 제안, 절대 막다른 길 금지
- 관련성, 날짜, 카테고리로 결과 필터/정렬

### 탐색 베스트 프랙티스
- 콘텐츠 미리보기 표시 (라벨만이 아닌)
- 필터링과 정렬 활성화
- 명확한 섹션 구분이 있는 "끝없는 스크롤" 지원
- 긴 리스트에 "맨 위로" 제공

## 교차 링킹 전략

모든 콘텐츠가 한 카테고리에 깔끔하게 들어가지는 않습니다. 교차 링킹이 이를 해결합니다.

### 기법
- **관련 항목**: "이런 것도 좋아하실 수 있어요" (콘텐츠 기반 유사도)
- **컨텍스트 링크**: 한 섹션에서 다른 섹션으로 딥링크 (예: 제품 → 리뷰)
- **빵 부스러기**: 깊은 콘텐츠에 대한 경로 표시 (주로 웹)
- **유니버설 검색**: 섹션과 무관하게 무엇이든 찾기
- **단축키/빠른 액션**: 자주 가는 목적지로 점프 (아이콘 롱 프레스, 스포트라이트 검색)

### 규칙
> 모든 콘텐츠는 적어도 2개의 다른 경로로 도달 가능해야 한다.

## 일반적인 IA 안티 패턴

| 안티 패턴 | 문제 | 해결 |
|----------|------|------|
| 조직도 내비 | 섹션이 사용자 태스크가 아닌 내부 팀을 반영 | 사용자 목표 중심으로 재디자인 |
| 잡동사니 서랍 | "더보기" 또는 "기타" 탭이 모든 것을 잡음 | 재정렬 또는 평탄화 |
| 깊은 중첩 | 콘텐츠 도달까지 4+ 레벨 | 검색/필터로 평탄화 |
| 모호한 라벨 | "리소스" vs "도구" vs "라이브러리" | 사용자와 라벨 테스트 |
| 중복 경로 | 같은 콘텐츠가 다른 이름으로 3 곳에 | 정규 위치 + 교차 링크 |
| 기능 우선 내비 | 사용자 목표가 아닌 기능으로 정리됨 | 재프레이밍: 사용자가 무엇을 하려는가? |
