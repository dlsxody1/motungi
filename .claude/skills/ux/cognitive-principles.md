# UX를 위한 인지 원칙

인터페이스 디자인에 적용된 행동과학. 인터페이스가 잘못된 느낌이 들거나 사용자가 어려움을 겪을 때 진단하기 위해 이 참고 자료를 사용하세요.

---

## 힉의 법칙 (Hick's Law)

**의사결정에 걸리는 시간은 선택지의 수와 복잡성에 따라 로그적으로 증가한다.**

### 적용 시점
- 사용자가 화면에서 망설이거나 멈춘다
- 의사결정 지점에서 전환율이 떨어진다
- 사용자가 "여기서 뭘 해야 하나요?" 라고 묻는다

### 솔루션
- 가시적 옵션 줄이기 (점진적 공개)
- 관련된 선택지를 카테고리로 그룹화
- 추천 옵션을 시각적 비중으로 강조
- <5%의 사용자가 필요로 하는 선택지 제거
- 스마트 기본값으로 의사결정 자체를 제거

### 예시
```
❌ 40개의 옵션이 보이는 설정 페이지
✅ 5개 카테고리로 그룹화된 설정, 각각 펼칠 수 있음

❌ 6개 등급 + 비교표가 있는 "플랜 선택"
✅ 3개 플랜, 하나가 "가장 인기"로 강조됨
```

## 피츠의 법칙 (Fitts's Law)

**타겟에 도달하는 시간은 타겟까지의 거리와 타겟의 크기의 함수이다.**

### 적용 시점
- 사용자가 탭/클릭 타겟을 놓친다
- 중요한 액션의 참여도가 낮다
- 모바일 사용자가 특정 버튼에서 어려움을 겪는다

### 솔루션
- 타겟 크기 키우기 (최소 44×44pt, 권장 48×48pt)
- 자주 사용되는 액션까지의 거리 줄이기
- 주요 액션을 엄지 친화적 영역에 배치
- 작은 시각 요소 주변에 패딩 추가 (큰 탭 영역, 작은 비주얼)
- 관련된 액션을 가까이 그룹화

### 예시
```
❌ 모달을 닫는 모서리의 작은 "X" 버튼
✅ 큰 "닫기" 버튼 또는 외부 탭으로 닫기 + 가시적 닫기 버튼

❌ 같은 크기로 저장 버튼 바로 옆의 삭제 버튼
✅ 저장 버튼은 크고 주요. 삭제는 더 작고, 분리되어 있고, 다른 색상.
```

## 인지 부하 이론 (Cognitive Load Theory)

**작업 기억은 약 4±1개의 정보 청크를 보유할 수 있다. 모든 요소가 주의를 두고 경쟁한다.**

### 세 가지 유형의 인지 부하
1. **본질적 부하**: 태스크 자체에 내재된 복잡성 (줄일 수 없음 — 세금 양식은 복잡함)
2. **외재적 부하**: 빈약한 디자인에서 오는 불필요한 복잡성 (줄일 수 있고, 줄여야 함)
3. **유관 부하**: 학습/이해에 쓰이는 노력 (도움이 됨 — 지원하라)

### 적용 시점
- 사용자가 압도된다고 느낀다
- 사용자가 중요한 정보를 놓친다
- 복잡한 폼에서 에러율이 증가한다
- 사용자가 작업 중간에 이탈한다

### 솔루션
- 현재 의사결정에 필요한 정보만 보여주기 (그 외에는 아무것도 없음)
- 복잡한 태스크를 단계로 나누기 (청킹)
- 점진적 공개 사용 — 필요할 때 세부사항
- 정보를 담지 않은 장식적 요소 제거
- 의사결정을 줄이기 위한 스마트 기본값 제공
- 회상이 아닌 인지 사용 (옵션을 보여주고, 사용자가 기억하게 만들지 마라)

### 예시
```
❌ 한 페이지에 15개 필드가 있는 체크아웃 폼
✅ 3단계 체크아웃: 배송 → 결제 → 검토

❌ 20개 지표를 동시에 표시하는 대시보드
✅ 3-5개 핵심 지표가 보이고, 나머지는 드릴다운으로 사용 가능
```

## 목표 그래디언트 효과 (Goal Gradient Effect)

**사람들이 목표에 가까워질수록 동기가 증가한다.**

### 적용 시점
- 사용자가 다단계 프로세스에서 이탈한다
- 완료율이 낮다
- 사용자가 중간에 흥미를 잃는다

### 솔루션
- 진행 인디케이터 표시 (Y 중 X 단계, 진행 바)
- 진행 바를 >0%에서 시작 (인공적 진전 — 예: 사용자가 아무것도 하기 전에 "프로필 20% 완료")
- 큰 태스크를 가시적인 마일스톤으로 나누기
- 마이크로 완료를 축하
- 완료된 것이 아니라 남은 것을 보여주기

### 예시
```
❌ 진행 인디케이터 없는 7단계 폼
✅ "4단계 중 3단계 — 거의 다 됐어요!"

❌ 적립 카드: "10번 사면 1번 무료" (0에서 시작)
✅ 적립 카드: 12개 슬롯, 2개 미리 찍힘 (2/12에서 시작 — 같은 10개 필요하지만, 더 가깝게 느껴짐)
```

## 피크-엔드 법칙 (Peak-End Rule)

**사람은 가장 강렬한 순간(피크)과 끝을 기준으로 경험을 판단하지, 평균으로 판단하지 않는다.**

### 적용 시점
- 사용자가 대부분 매끄러운 플로우에도 불구하고 부정적 경험을 보고한다
- NPS 점수가 예상보다 낮다
- 태스크 완료 후 사용자가 돌아오지 않는다

### 솔루션
- 가장 고통스러운 순간을 식별하고 수정 (피크 부정 → 가장 큰 ROI)
- 즐거운 마무리 만들기 (확인, 축하, 명확한 다음 단계)
- 당신의 업셀이 아닌 사용자의 성취로 끝내기
- 평균 경험이 "괜찮"더라도 최악의 마찰 지점을 매끄럽게

### 예시
```
❌ 구매 후: 즉시 "이런 것도 좋아하실 수 있어요..." 표시
✅ 구매 후: "다 됐습니다! 주문 #123 확인됨." → 명확한 다음 단계 → 그 다음 추천

❌ 온보딩이 설정의 벽으로 끝남
✅ 온보딩이 사용자의 첫 성공적인 액션으로 끝남 ("첫 워크스페이스가 준비됐어요!")
```

## 밀러의 법칙 (Miller's Law)

**평균적인 사람은 작업 기억에 7±2개의 항목을 보유할 수 있다. (현대 연구는 복잡한 항목에 대해 4±1개를 제시한다.)**

### 적용 시점
- 내비게이션이 많은 최상위 항목을 가진다
- 리스트가 구조 없이 끝없게 느껴진다
- 사용자가 이전 화면의 옵션을 기억할 수 없다

### 솔루션
- 내비게이션을 5개 항목(모바일) 또는 7개 항목(데스크톱)으로 제한
- 정보를 3-5개의 그룹으로 청킹
- 인지된 항목 수를 줄이기 위해 시각적 위계 사용
- 화면 간 중요한 컨텍스트 유지 (사용자가 외우게 만들지 마라)
- 긴 리스트를 청킹하기 위해 헤딩 사용

## 폰 레스토프 효과 (Von Restorff Effect, 분리 효과)

**주변과 시각적으로 구별되는 항목이 더 잘 기억되고 행동을 유발할 가능성이 높다.**

### 적용 시점
- 주요 액션이 충분한 클릭을 받지 못한다
- 사용자가 중요한 경고를 놓친다
- 핵심 정보가 페이지에서 사라진다

### 솔루션
- 화면당 시각적으로 구별되는 한 가지 만들기 (주요 액션)
- 주요 요소를 분리하기 위해 색, 크기, 또는 위치 사용
- 모든 것을 "두드러지게" 만들지 마라 — 모든 게 굵으면, 아무것도 굵지 않다
- 강조 색상은 인터랙티브/실행 가능한 요소에만 예약

### 예시
```
❌ 같은 스타일의 세 버튼: [취소] [임시 저장] [발행]
✅ [취소]와 [임시 저장]을 텍스트/고스트 버튼으로, [발행]을 채워진 주요 버튼으로

❌ 모든 섹션 헤더가 밝은 파란색에 굵게
✅ 크기/굵기로 콘텐츠 위계, 색상은 액션에 예약
```

## 야콥의 법칙 (Jakob's Law)

**사용자는 대부분의 시간을 다른 앱에서 보낸다. 그들은 당신의 앱이 똑같이 작동하기를 기대한다.**

### 적용 시점
- 표준 태스크에 대해 새로운 인터랙션을 디자인하고 있다
- 내비게이션이나 일반 패턴에서 "혁신"하고 싶은 유혹이 든다
- 사용자가 표준 컴포넌트를 대체한 커스텀 컴포넌트에서 어려움을 겪는다

### 솔루션
- 표준 태스크에는 플랫폼 표준 패턴 사용 (iOS 탭 바, Android 머티리얼 내비)
- 커스텀 패턴은 커스텀 기능에만
- 벗어나야 한다면, 커스텀 패턴이 명백히 더 나아야 한다 (단지 다른 게 아닌)
- 경쟁자와 카테고리 리더 패턴을 연구 — 사용자는 그것들로부터 기대를 가지고 온다

### 예시
```
❌ 소셜 앱에 탭 바 대신 커스텀 햄버거 메뉴 ("더 깔끔하니까")
✅ 표준 탭 바 — 사용자는 다른 모든 소셜 앱에서 어떻게 작동하는지 정확히 안다

❌ 새로운 인터랙션의 커스텀 날짜 선택기 위젯
✅ 네이티브 날짜 선택기 (사용자가 알고, 기본적으로 접근 가능함)
```

## 도허티 임계값 (Doherty Threshold)

**응답 시간이 <400ms일 때 생산성이 급증한다. 그 이상에서는 사용자가 지연을 인지하고 흐름을 잃는다.**

### 적용 시점
- 시스템이 정상 작동에도 불구하고 "느리게" 느껴진다
- 사용자가 첫 번째가 등록되지 않았다고 생각해서 더블 탭/클릭을 한다
- 기술적으로 올바른 화면에서 사용자 참여가 떨어진다

### 솔루션
- 입력 피드백에 <100ms 목표 (버튼 누름 시각 변화)
- 시스템 응답에 <400ms 목표 (내비게이션, 데이터 표시)
- >400ms 작업에는 낙관적 UI 사용
- 콘텐츠 로딩에 즉시 스켈레톤 화면 표시
- 전환을 위한 애니메이션은 300ms 이내에 완료

## 미적-사용성 효과 (Aesthetic-Usability Effect)

**사용자는 객관적으로 그렇지 않을 때조차 미적으로 즐거운 디자인을 더 사용하기 쉽다고 인지한다.**

### 적용 시점
- 사용자가 인터페이스를 "사용하기 어렵다"고 보고하지만 사용성 지표는 괜찮다
- 비슷한 기능에도 불구하고 사용자가 경쟁자 제품을 선호한다
- 신규 사용자와 신뢰를 쌓으려고 한다

### 솔루션
- 시각적 다듬음은 사소한 사용성 이슈에 대한 관용을 만든다
- 첫인상은 비례를 넘어 중요하다 — 진입 화면의 시각 품질에 투자하라
- 단, 미적 디자인은 근본적으로 망가진 플로우를 보상할 수 없다
- 미와 사용성을 함께 — 표준이지, 미 또는 사용성 중 하나가 아니다

## 직렬 위치 효과 (Serial Position Effect)

**사람은 리스트의 첫 번째(초두 효과)와 마지막(최신 효과) 항목을 중간 항목보다 더 잘 기억한다.**

### 적용 시점
- 내비게이션 순서를 디자인하고 있다
- 리스트, 메뉴, 또는 온보딩 단계에서 항목을 정렬한다
- 페이지의 콘텐츠를 구조화한다

### 솔루션
- 가장 중요한 내비게이션 항목을 처음과 마지막에 배치 (홈은 왼쪽, 프로필은 오른쪽)
- 긴 리스트에서, 핵심 콘텐츠를 앞에 배치하고 강한 마무리를 추가
- 중간 항목은 알아채려면 추가 시각적 강조가 필요
- 온보딩의 경우: 첫 번째와 마지막 단계가 가장 강한 인상을 남긴다

## 자이가르닉 효과 (Zeigarnik Effect)

**사람은 완료된 태스크보다 미완성 태스크를 더 잘 기억한다.**

### 적용 시점
- 저장/이어하기 경험을 디자인한다
- 사용자가 프로필이나 프로세스를 완성하도록 유도한다
- 알림 전략을 디자인한다

### 솔루션
- 미완성 항목을 두드러지게 표시 (진행 바, "이전에 하던 곳부터 이어가기")
- 부분 진행을 자동 저장 (사용자의 작업을 절대 잃지 않기)
- 미완성을 동기로 사용: "프로필이 60% 완료됨"
- 단, 무기화하지 마라 — 잔소리는 동기가 아니라 불안을 만든다

## 원칙 선택 가이드

| 사용자 문제 | 주요 원칙 | 보조 원칙 |
|------------|----------|----------|
| 사용자가 망설인다 | 힉의 법칙 | 인지 부하 |
| 사용자가 타겟을 놓친다 | 피츠의 법칙 | 폰 레스토프 |
| 사용자가 압도된다 | 인지 부하 | 밀러의 법칙, 힉의 법칙 |
| 사용자가 중도 이탈한다 | 목표 그래디언트 | 자이가르닉, 피크-엔드 |
| 사용자가 부정적으로 기억한다 | 피크-엔드 법칙 | 미적-사용성 |
| 사용자가 커스텀 패턴에 혼란스러워한다 | 야콥의 법칙 | 인지 부하 |
| 주요 액션이 무시된다 | 폰 레스토프 | 피츠의 법칙, 직렬 위치 |
| 시스템이 느리게 느껴진다 | 도허티 임계값 | 목표 그래디언트 |
| 사용자가 돌아오지 않는다 | 미적-사용성 | 피크-엔드, 야콥의 법칙 |
| 사용자가 프로필을 완성하지 않는다 | 자이가르닉 효과 | 목표 그래디언트 |
